Features

What it includes.

Live monitoring

CPU activity, Apple Silicon core usage, memory pressure, thermal readings, power, battery state, and fan RPM in one native interface.

Fan control

Optional helper-backed fan control with fixed modes, smart curves, and wake re-apply behavior.

Touch Bar

Custom layouts for status, weather, hardware, launcher, and command items on supported Touch Bar Macs.

Menu bar utility

Quick readings and controls without opening the full dashboard.

Native macOS app

Built in Swift for macOS instead of wrapped from the web.

Privacy

Built to stay local.

Open source

The code is published in the open so you can inspect how Core-Monitor works and what it talks to.

Local sensor reads

Hardware readings come from the Apple SMC and standard macOS APIs. Core monitoring stays on your Mac.

Optional weather

The Touch Bar weather item uses Apple WeatherKit and location access. Remove that item if you do not want local weather in the Touch Bar.

Standard macOS services

Launch at login uses macOS login items, and fan control uses the bundled local helper only when you enable that path.

First-run checklist

What to check after first launch.

Open the app

Launch Core-Monitor from the downloaded release or from Xcode if you built from source.

Allow the app if macOS asks

Use the normal macOS first-run prompts if your system asks for confirmation before opening the app.

Enable fan writes only if you want them

You can use the monitoring features without the helper. Fan writes depend on the helper path being available.

Enable Launch at Login if you want it nearby

Core-Monitor uses the standard macOS login item approval flow.
